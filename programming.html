Phases of Computer Program Development:
  • Phase 1: Understand the problem.
  • Phase 2: Design an algorithm to solve the problem.
  • Phase 3: Write the program code in appropriate language based on the algorithm.
  • Phase 4: Enter the program and save.
  • Phase 5: Execute the program.
  • Phase 6: Test the program, find errors and debug, then retest.
  • Phase 7: Document and evaluate the program.
Programming Structures:
  • Sequential structure: Top-down execution
  • Decision structures: if/then/else
  • Iterative (Looping): While loop: continue looping as long something is true.
  • Repeat/Until Loop: continue looping until something is true
  • For loop: loop a given number of times.
  • Recurrsive Structures: Repeat a set of instructions as a subtask of itself.
  • Repetition Continues Until: Degenerative case, which is a termination conditions, is met (degenerative case is also called a "base case")
Two Classic Programming/Database Problems:
  • Sorting: Decscending order/ Ascending order.
  • Sorting numbers.
  • Sorting strings (in accordance with ASCII values)
  • Searching: Find specific data that meets a certain criteria
  • In database:
  • Filters: temporary search
  • Queries: search criteria are saved so they dont have to be recreated.
  • Two Popular Search Techniques:
  • Sequential SearchL Search from top to bottom or bottom to top, one item at a time.
  • Binary Search: only works if data is SORTED first. Start in the middle, then go to half of that, etc.
Two Classic Program Goals:
  • Efficiency: Program takes minimal time and/or resources
  • Correctness: Program runs in expected ways and works for all expected conditions.
Software Testing and Verification:
  • Kinds of Test:
  • Acceptance Testing:Verifying whether the whole system works as intended.
  • Integration Testing:Ensuring that software components or functions operate together.
  • Unit Testing:Validating that each software unit performs as expected. A unit is the smallest testable component of an application.
  • Fuctional Testing:Checking functions by emulating business scenarios, based on functional requirements. Black-box testing is a common way to verify functions.
  • Performance Testing:Testing how the software performs under different workloads. Load testing, for example, is used to evaluate performance under real-life load conditions.
  • Regression Testing:Checking whether new features break or degrade functionality. Sanity testing can be used to verify menus, functions and commands at the surface level, where there is no time for a full regression test.
  • Stress Testing:Testing how much strain the system can take before it fails. Considered to be a type of non-functional testing.
  • Usability Testing:Validating how well a customer can use a system or web application to complete a task.
  • Software testing is done to detect:
  • Architechtural flaws
  • Poor design decisions
  • Invalid of incorrect functionality
  • Security vulnerabilities
  • Scalability issues
  • Testing best practices:
  • Continuous testing:A software development process in which applications are tested continuously throughout the entire software development life cycle.
  • Configuration Management:The process of maintaing systems, such as computer hardware in a desired state.
  • Servic Virtualization:A method to emulate the behavior of specific componenets in heterogenerous component based applications.
  • Defect or "bug tracking":The process of logging and monitoring bugs or errors during software testing.
  • Metrics and Reporting:A standard of measure to a degree which a software system or process possesses some property.
