Chapter 7: Software Engineering

Term

Definition
Software Engineering: A systematic engineering approach to software development
Integrated Development Environment (IDE): Applications that facilitates on the development of other applications.
Association for Computing Machinery (ACM): Purpose of ACM: A society for Computer Scientists, Software Engineers, and tech enthusiasts. Benefits of ACM membership: You become part of the dynamic changes that are transforming our world by shaping the future of computers. Student ACM membership: Student ACM memberships cost $19, which is cheaper than non-students.
Institute of Electrical and Electronics Engineers (IEEE): Purpose of IEEE: To foster technological innovation and excellence for the benefit of humanity. Benefits of IEEE Membership: The ability to network with professionals, being made aware of scholarships, research programs, internships, and other employment opportunities. Student IEEE Membership: Must be enrolled in classes 50% academically as a registered undergrad or grad student in a IEEE designated field.
Software Life Cycle: Development Phase Where the development team turns product specifications and business requirements inot code that makes the product. Requirements Analysis Determining the feasibility of creating the product, revenue potential, cost of production and the needs of the users. Design: Where the original plan and vision is elaborated into the basic structure of the software, including system design, programming language,templates, platforms, and application security. Implementation: When your software is delivered to your intended user. Testing: Before getting the software product out the door, its important to have a team test it to make sure it is functioning properly and does what its meant to do.
Modular Implementation: The process of subdividing a computer program into seperate sub-programs.
Unified Modeling Language (UML): A standardized modeling language enabling developers to specify, visualize, construct and document artifacts of a software system.
Use Case: A software testing technique that helps identify the test cases that encompass the entire system on an action-to-action basis.
Use Case Diagram: Used to represent the dynamic behavior of a system, it encapsulates the systems functionality by incorporating use cases, actors, and their relationships.
Data Relationships: One-to-One: A relationship where a record in one table is associated with exactly one record in another table relationship where a record in one table is associated with exactly one recorded in another table. One-to-Many: Occurs when a parent record in one table can potentially refernce several child records in another table. Many-to-Many: A relationship between tables in a database when a parent row in one table contains several child rows in the second table, and vice versa.
Design Pattern: A general, reusable solution of how to solve a common problem when designing an application or system.
Software Quality Assurance (SQA): A set of activities for ensuring quality in software enginerring processes.
Software Testing: Glass Box Testing: A testing technique that examins the program structure and derives test data from the program logic/code. Black Box Testing: A type of software testing in which the functionality of the software is not known. Beta Testing: A beta version of the software, whose feedback is needed, is released to a limited number of end-users of the product to obtain feedbaack on the product quality. Alpha Testing: It is the first round of testing a new product or software solution goes through, aplha testing is concerned with finding any possible issues, bugs, mistakes, before progressing to user testing or market launch.
Documentation: User Documentation: The doccumentation for a product or service provided to the end users. System Documentation: It is comprised of detailed language, illustrations and photos that help different people understand the software, and it is essential reference material. Technical Documentation: The umbrella term that ecompasses all written documents and materials dealing with software product development.
Intellectual Property Law: Software License: A contract between the entity that created and supplied an application, underlying source code, or related product and its end user. Software Disclaimer: The software developers formal notice to the user of its software that the developer is denying any and all liability for for any damages arising out of using the software as well as dening any implied warranties. Best Defenses Against Software Liability Claims: Argue that the plaintiffs injury is unrelated to the product.
